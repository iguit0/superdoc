THE WORLD WE LIVE IN

SuperDoc gives AI agents 193 tools
to edit .docx documents

User prompt
→
LLM picks tools
→
CLI executes
→
Document edited

The LLM never touches the document.

It reads JSON Schema tool definitions, picks the best match for the user's task, and outputs a structured function call. Your code runs it. This is how Claude Code, Cursor, and every AI agent works.

// SuperDoc agent loop (simplified) while (hasToolCalls) { response = llm(prompt, tools) // LLM picks result = cli(response.tool) // CLI runs prompt.add(result) // feed back }
THE RISK

This chain has four moving parts.

01

Model

GPT-5.4 ships. Claude updates. Tool selection changes overnight.

02

Tool definitions

Rename a tool. Rewrite a description. Every agent breaks.

03

CLI

Refactor execution. Change response format. New validation rules.

04

System prompt

Tweak instructions. Model picks different tools entirely.

Any one changes silently. The customer's $25M becomes $150M and nobody notices.

You edit .docx files using SuperDoc tools. Be efficient -- minimize tool calls. IMPORTANT: Never pass "doc" or "sessionId"... ## Tool Categories ## Recommended Workflow ## superdoc_search -- targeting ## superdoc_mutations -- valid step ops ## Multi-action tools ## Common Errors

You help users edit .docx documents using the provided tools. Task: {{task}}
